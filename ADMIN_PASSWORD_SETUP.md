# 管理者パスワード設定ガイド

`/setup`ページを保護するためのパスワード設定方法です。

## 🔐 設定手順

### 1. 環境変数を設定

`.env.local`ファイルに以下を追加：

```bash
# 管理者パスワード（/setupページのアクセスに必要）
ADMIN_PASSWORD=your_secure_password_here
```

**重要**: 
- 強力なパスワードを設定してください（最低8文字以上推奨）
- このパスワードは`.env.local`にのみ保存し、Gitにコミットしないでください
- 本番環境（Vercel等）でも環境変数として設定してください

### 2. 開発サーバーを再起動

環境変数を追加した後、開発サーバーを再起動してください：

```bash
npm run dev
```

### 3. 動作確認

1. ブラウザで `/setup` にアクセス
2. パスワード入力画面が表示されることを確認
3. 正しいパスワードを入力して認証
4. 管理画面が表示されることを確認

## 🔒 セキュリティ機能

- **パスワード検証**: サーバー側（API Route）でパスワードを検証
- **セッション管理**: 認証成功後、localStorageに認証トークンを保存
- **ログアウト機能**: ヘッダーに「ログアウト」ボタンを追加

## ⚠️ 注意事項

- **localStorageベースの認証**: 現在の実装はlocalStorageを使用しています。より強固なセキュリティが必要な場合は、JWTやセッション管理システムの導入を検討してください。
- **パスワードの管理**: パスワードは環境変数として管理し、コードに直接書かないでください。
- **本番環境**: Vercel等の本番環境でも、環境変数として`ADMIN_PASSWORD`を設定してください。

## 🚀 本番環境での設定（Vercel）

1. Vercel Dashboardにアクセス
2. プロジェクトを選択
3. Settings → Environment Variables
4. `ADMIN_PASSWORD`を追加
5. デプロイを再実行

## 💡 より強固なセキュリティが必要な場合

以下のような追加対策を検討してください：

1. **JWT認証**: より安全なトークンベースの認証
2. **IP制限**: 特定のIPアドレスのみアクセス可能にする
3. **2要素認証**: パスワード + 追加の認証要素
4. **レート制限**: パスワード試行回数の制限
